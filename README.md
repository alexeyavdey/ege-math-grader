# Тестовое задание: сервис генерации ЕГЭ по математике с AI-проверкой

Привет! Это небольшой сервис на FastAPI, который генерирует задания ЕГЭ по
математике (квадратные уравнения) и проверяет ответы учеников с помощью LLM.
Код намеренно **неполный и с дефектами** — твоя задача довести его до рабочего
состояния и расширить.

Мы оцениваем не «сколько кода написано» (с AI-ассистентом кода будет много), а
**твои инженерные решения, умение проверять результат и ловить ошибки** — в том
числе ошибки LLM. Пользоваться Claude Code / любым AI можно и нужно: в реальной
работе ты будешь работать именно так.

Ориентир по времени — **половину рабочего дня (2–3 часа)**. Доводить до продакшена
не нужно; важнее правильные приоритеты и корректность.

---

## Что внутри

```
app/
  main.py        FastAPI: POST /tasks (генерация), POST /tasks/{id}/grade (проверка)
  generator.py   генерация квадратного уравнения
  verifier.py    независимая проверка корректности  ← НЕ РЕАЛИЗОВАН (заглушки)
  grader.py      проверка ответа ученика через LLM
  llm.py         интерфейс LLM + FakeLLM (офлайн-заглушка)
  store.py       in-memory хранилище
  models.py      схемы API
tests/           pytest (часть тестов падает / пропущена — см. ниже)
```

## Как запустить

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

uvicorn app.main:app --reload      # сервис на http://127.0.0.1:8000 (/docs — Swagger)
pytest -q                          # тесты
```

Если запустишь тесты сейчас, то увидишь: часть зелёные, один падает, два
пропущены. Это нормально — это часть задания.

## Что нужно сделать

**1. Разберись и почини.**
Сделай тест-сьют зелёным. В README напиши, **в чём была причина** падающего теста
и почему так делать нельзя. (Чинить нужно причину, а не подгонять тест.)

**2. Реализуй независимую проверку корректности (`app/verifier.py`).**
Истинные корни должны вычисляться из условия задачи символически (sympy), а не
браться на веру. Используй верификатор в двух местах:
   - **в генерации** — не отдавать ученику задание, у которого ключ не сходится;
   - **в проверке ответа** — вердикт «верно / неверно» должен опираться на
     посчитанную истину, а не на мнение LLM.
   Включи и дополни тесты в `tests/test_verifier.py`.

**3. Подключи настоящую LLM (`app/llm.py`).**
Реализуй провайдера (OpenAI / Anthropic / Gemini — любой) по интерфейсу `LLM` и
используй его в `grader.py` для **человекочитаемого объяснения**, почему ответ
неверный (а не для вынесения вердикта). `FakeLLM` оставь провайдером по умолчанию,
чтобы тесты и запуск работали офлайн без ключей. Ключ читай из переменной
окружения; **не коммить секреты**.

**Если останется время (по желанию):** второй тип задания; обработка некорректного
ввода ученика; эндпоинт, который гарантированно отдаёт только провалидированное
задание; что-то на свой вкус, что считаешь важным.
